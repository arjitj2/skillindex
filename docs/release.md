# Release Process

Skill Index ships from `skillindex.app` as a signed and notarized universal macOS DMG. GitHub Releases store the release assets and updater metadata, but users should not need to visit GitHub to install or update the app.

The `Release` workflow repeats lint, typecheck, tests, and build in GitHub Actions before signing, notarizing, smoke-launching, and publishing tagged releases.

Windows and Linux are coming soon. Do not add public download links for those platforms until their packaged builds have CI smoke tests equivalent to the macOS release workflow.

## Release Artifacts

The production macOS release publishes:

- `Skill-Index-<version>-universal.dmg` for the public website download
- `Skill-Index-<version>-universal.zip` for Electron auto-update
- `latest-mac.yml` for update metadata
- blockmap files generated by electron-builder

The app identity is `app.skillindex.desktop`, the product name is `Skill Index`, and production releases use the standard build flavor.

## Required GitHub Secrets

The `Release` workflow expects these repository secrets:

- `MAC_CERTIFICATE`: base64-encoded Developer ID Application `.p12` export
- `MAC_CERTIFICATE_PASSWORD`: password for the `.p12` export
- `APPLE_ID`: Apple Developer account email
- `APPLE_APP_SPECIFIC_PASSWORD`: app-specific password for notarization
- `APPLE_TEAM_ID`: Apple Developer Team ID

Create the app-specific password from the Apple account used for notarization. Export the Developer ID Application certificate from Keychain Access as a password-protected `.p12`, then base64-encode it before storing it in GitHub:

```bash
base64 -i DeveloperIDApplication.p12 | pbcopy
```

## Creating A Release

1. Update `package.json` to the release version.
2. Run local validation:

```bash
pnpm install --frozen-lockfile
pnpm lint
pnpm typecheck
pnpm test -- --maxWorkers=4
pnpm build
```

3. Optionally build unsigned local artifacts:

```bash
pnpm dist:mac:unsigned
```

4. Commit the version change.
5. Create and push a version tag:

```bash
git tag v0.1.0
git push origin v0.1.0
```

The tag starts `.github/workflows/release.yml`. The workflow builds the universal macOS app, signs it, notarizes it, verifies Gatekeeper assessment, launches the packaged app with an isolated data directory, and publishes the GitHub Release assets.

## Signing Smoke Test

Before publishing the first real release, run the `Release` workflow manually from the GitHub Actions tab. Manual runs build the same universal macOS app, import the Developer ID certificate, sign, notarize, staple, verify Gatekeeper assessment, and smoke-launch the packaged app.

Manual runs do not publish GitHub Release assets. Only `v*.*.*` tag pushes run the final publish step.

## Auto-Updates

Production packaged builds check GitHub release metadata shortly after startup and then every five minutes. Dev builds, dev-alpha builds, unpackaged builds, and processes launched with `SKILL_INDEX_DISABLE_AUTO_UPDATE=1` do not check for updates.

When an update has downloaded, Skill Index shows an `Update` button in the sidebar footer next to the last scan time. Clicking it restarts the app and installs the downloaded update. The app does not show a native update modal.

## Website Download

Point `skillindex.app` at the latest stable universal DMG. The primary public copy should say Mac is available now and Windows/Linux are coming soon.
